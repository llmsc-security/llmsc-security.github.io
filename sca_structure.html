Result Schema Structure

Understanding the OpenSCA JSON output format

{} Root Object
{} meta
[] results
{} component object
[] vulnerabilities
Root
Object
The top-level container for the scan results. Contains metadata about the scan execution and the array of findings.
{ "meta": { ... }, "results": [ ... ] }
meta
Object
Contains information about the scanner execution, including timestamp, tool name, and any global warnings.
"meta": { "generated_at": "2025-12-22T06:01:39Z", "scanner": "custom-sca-osv-baseline", "source": "sca_result.json" }
results
Array<Object>
A list of all scanned components (dependencies). This includes both safe and vulnerable components.
"results": [ { "component": "axios", "vulnerable": true, ... } ]
component (Item)
Object
Represents a single dependency found in the project.
{ "component": "axios", "ecosystem": "npm", "version_constraints": ["^0.26.0"], "vulnerable": true, "vulnerabilities": [ ... ] }
vulnerabilities
Array<Object>
Detailed list of security flaws associated with a specific component. If the component is safe, this array is empty.
"vulnerabilities": [ { "id": "GHSA-42xw-2xvc-qx8m", "summary": "Denial of Service in axios", "cves": ["CVE-2019-10742"], "cvss": "CVSS:3.0/AV:N/AC:L...", "cwes": [ { "id": "CWE-20", "name": "Improper Input..." } ] } ]
